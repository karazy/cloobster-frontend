Release Notes - Was gibt es Neues?

Frontend Version 1.9

  • App Features konfigurieren - Funktionen einer Location können einzeln an- und ausgeschaltet werden, so dass diese in der App nicht mehr eingeblendet werden.
  • Produkte bestellen ausschalten - eizelne Produkte können als nicht bestellbar markiert werden.

Frontend Version 1.8

  • Feedback Reporting - detaillierte Auswertungen zu gesendetem Feedback von App Nutzern
  • Umsatz Reporting - Beträge von abgeschlossenen Bestellungen nach Tag summiert
  • kleinere Fehler behoben

Frontend Version 1.7

  • App Dashboard Konfiguration - die Inhalte der Startseite direkt selber bestimmen
  • Bearbeiten von übersetzten Inhalten bei Infoseiten überarbeitet - gleichzeitig mehrere Übersetzungen sehen, während man Felder bearbeitet
  • Web links für Infoseiten - Infoseiten können einen Link zu einer Website enthalten diese werden in der App geöffnet, um z.b. auf lokale Angebote und Partner zu verlinken
  • kleinere Fehler behoben

Frontend V1.6.3 und Cockpit V2.1

Frontend

  • kleinere Fehler behoben

Cockpit

  • Anzeige von inaktiven Check-Ins mittels Button in der unteren Aktionsleiste. Ein Check-In gilt als inaktiv, wenn seit 24 Stunden keine Aktionen stattgefunden hat. In regelmäßigen Abständen (alle 12 Stunden) wird automatisch auf inaktive Check-Ins hingewiesen.
